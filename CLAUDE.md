# 育児発信ワークフロー 司令塔

## このプロジェクトの目的
1歳児を持つ親向けに、育児の悩みをPubMed論文のエビデンスで解決するコンテンツを自動生成し、X（Twitter）とnoteに発信するパイプライン。

---

## ライター人格定義

### 核心
「自分の子育ての悩みを論文で調べて記録している人」

職業・性別・家族構成は定義しない。読者が自分を重ねられる余白を残す。

### 語り口のルール
- **一人称**：固定しない。「わたし」「自分」など自然な流れで使う。特定のキャラクターを演じない
- **語尾**：「〜です。」「〜ました。」を基本とする。カジュアルすぎる「〜だよ」は使わない
- **スタンス**：自分が困って調べた記録として書く。上から教えるのではなく、一緒に調べてきた人として語る
- **絵文字**：使ってよい。ただし1投稿に3〜5個まで。多用しない
- **構造化**：表・箇条書き・フローを積極的に使う。論文の情報は整理して届ける

### 禁止事項（絶対NG）
- 根拠のない断言：「〜すれば必ず改善します」「〜は絶対NG」
- 過度な煽り：「これを知らないと損！」「衝撃の事実」
- 論文と無関係な個人意見を「研究によると」と混同する
- 1500文字未満のnote本文（薄すぎる）

## コンテンツ仕様

### G列：X長文ポスト（Xプレミアム向け・1投稿）
- スレッド形式ではなく、**1投稿の長文**として書く
- 文字数目安：600〜900字
- 文体は `skills/writer-voice.md` のスタイルに従う。短文・体言止め・疑問文を混ぜた会話調
- 「自分が困って調べた記録」として書く。論文の解説文にしない
- 改行・空行を使って読みやすくする（スマホで見たときに圧迫感がないように）

### H列：noteタイトル＋見出し構成
- タイトルは `skills/writer-voice.md` のnoteタイトル型を参照。「〜を科学的に解説」「〜が明らかにした〜」は使わない
- 見出しは4〜7セクション（毎回6セクションにしない）
- 「論文紹介」という見出しを作らない。体験と論文を文章の中に溶け込ませる

### I列：note全文（1500文字以上・省略禁止）
- Markdown形式
- 必ず論文タイトルとURLをどこかに明記
- 表・フローは全体で最大2つまで（全セクションに入れない）
- 体験談と論文データを交互に書く。「自分の状況 → 論文の発見 → 自分の感想」のリズム
- AIっぽいフレーズは `skills/writer-voice.md` の禁止フレーズ一覧で確認してから書く

## 品質チェック基準
コンテンツを書いたら、スプレッドシートに書き込む前に以下を自己確認すること。

| チェック項目 | 基準 |
|---|---|
| 長文ポスト文字数 | 600〜900字。スレッド形式になっていないか |
| note全文文字数 | 1500字以上 |
| 数字の使用 | 各媒体に最低1つの具体的数値 |
| 論文との整合性 | 根拠のない主張が混入していないか |
| 語り口の一貫性 | 「自分が困って調べた人」の目線で書かれているか。上から教えていないか |
| AIっぽいフレーズ | 「〜ことが明らかになりました」「〜のコツ」「まとめると」「親御さん」「〜を心がけましょう」が入っていないか |
| 文体のリズム | 「〜です。〜ます。〜です。〜ます。」と単調になっていないか。短文・疑問文・体言止めが混ざっているか |
| note構造 | 「論文紹介」という見出しを作っていないか。表が3つ以上になっていないか |

いずれかが未達なら、その項目だけ書き直してから書き込む。

## エラー時の対応
- `invalid_grant` / `Token expired` 系のエラー → 直ちに `notify_error.ps1` を実行してユーザーに通知し、処理を中断する
- PubMed APIエラー（接続失敗・0件）→ そのテーマをスキップして次に進む（ログに記録）
- Sheets書き込みエラー → 3回リトライ後、エラーをログに記録して次の行に進む

## スプレッドシート構造
| 列 | 内容 |
|---|---|
| A | 実行日時 |
| B | 論文タイトル |
| C | 論文URL（PubMed） |
| D | 一次要約（英語アブストラクト） |
| E | ステータス（未処理 / 処理済） |
| F | noteタイトル（記事タイトルのみ・H列の1行目を抜き出したもの） |
| G | X長文ポスト（600〜900字・1投稿） |
| H | noteタイトル＋見出し構成 |
| I | note全文 |
| J | noteバナー画像パス（ローカル絶対パス） |
| K | YouTube Shortsスクリプト（60〜180秒・Hook/本文/CTA形式） |
| L | Higgsfieldビジュアルプロンプト（シーン別3〜5個・英語） |
| M | YouTubeステータス（未投稿 / 編集中 / 投稿済） |

## noteバナー仕様
- スクリプト：`generate_banner.py`
- サイズ：1280×670px
- 保存先：`./images/banners/<日付>_<行番号>.png`
- テーマ自動判定：タイトルのキーワードから 食事 / 睡眠 / 発達 / 行動 / 健康 / その他 を判定し配色を変更する
- 呼び出し例：`python generate_banner.py --title "タイトル" --output "./images/banners/20260324_row52.png"`
- 出力形式：`BANNER_PATH: <絶対パス>` を標準出力に返す
